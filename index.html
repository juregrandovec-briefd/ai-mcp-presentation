What Are AI Coding Agents?

Autonomous systems that execute complete development workflows

Key Characteristics:

  • Execute multi-step workflows
  • Make autonomous decisions
  • Use tools and APIs
  • Iterate until task complete
  • Ask for approval at each step

MCP Setup - Configuration

Configuration File Location:
~/Library/Application Support/Claude/claude_desktop_config.json
Basic Structure:
{
"mcpServers": {
"github": {
"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-github"],
"env": {
"GITHUB_PERSONAL_ACCESS_TOKEN": "your_token_here"
}
}
}
}

API tokens required for each service

TIMING: 1.5 minutes
  • Setup is straightforward - edit a JSON configuration file
  • For Claude Code: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Structure is simple: "mcpServers" object. Each server gets a name (like 'github'), specify command ('npx'), args (MCP server package), environment variables (API token)
  • You'll need API tokens for each service - GitHub personal access token, Slack bot token, etc. Once set up, it just works
  • Don't worry if it looks intimidating - documentation walks through step by step

MCP Setup - Multiple Services

{
"mcpServers": {
"github": {
"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-github"],
"env": {"GITHUB_PERSONAL_ACCESS_TOKEN": "your_github_token"}
},
"slack": {
"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-slack"],
"env": {
"SLACK_BOT_TOKEN": "your_slack_bot_token",
"SLACK_TEAM_ID": "your_team_id"
}
},
"postgres": {
"command": "npx",
"args": ["-y", "@modelcontextprotocol/server-postgres"],
"env": {"DATABASE_URL": "postgresql://user:pass@localhost/db"}
}
}
}
Setup time per MCP: ~5-10 minutes
TIMING: 1 minute
  • Here's a real example with multiple services. Notice the pattern - just repeating the same structure for each
  • GitHub: personal access token. Slack: bot token + team ID. Postgres: database connection string. Same pattern, different credentials
  • Bottom box - reassure: Each MCP takes 5-10 minutes to set up (mostly getting API tokens). Once configured, never touch it again

Complete Prompt Structure

7-Part Template:

1. Context: What's the situation?

2. Task: What needs to be done?

3. Constraints: Any limitations?

4. Output format: How should it respond?

5. Expected Behavior: Describe edge cases

6. Success Criteria: Define "done"

7. Reasoning Request: Ask agent to explain approach first
